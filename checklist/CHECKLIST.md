# 手動セキュリティチェックリスト

ツールでは検出できない項目を手動で確認するためのリスト。
OWASP Top 10 / IPA「安全なウェブサイトの作り方」をベースにしています。
リリース前・大きな変更後に一通り確認してください。

## 認証・セッション

- [ ] パスワードはハッシュ化(bcrypt/argon2等)して保存している — 漏洩時に平文が流出しないため
- [ ] ログイン試行回数の制限やレートリミットがある — 総当たり攻撃を防ぐため
- [ ] セッションIDはログイン成功時に再生成している — セッション固定攻撃を防ぐため
- [ ] ログアウトでセッションを確実に破棄している — 共有端末での乗っ取りを防ぐため
- [ ] 重要操作(パスワード変更等)には再認証を要求している — セッション乗っ取り時の被害を抑えるため

## 認可・権限

- [ ] URLやIDを書き換えても他人のデータにアクセスできない(IDOR対策) — 権限チェックの漏れが最多の脆弱性のため
- [ ] 管理機能はロールで保護され、一般ユーザーから到達できない — 権限昇格を防ぐため
- [ ] APIも画面と同じ権限チェックをしている — 画面側だけの制御は容易に迂回されるため

## 入力・出力

- [ ] SQLはプレースホルダ(パラメータ化)を使っている — SQLインジェクションを防ぐため
- [ ] ユーザー入力を表示する箇所はエスケープしている — XSSを防ぐため
- [ ] ファイルアップロードは拡張子・サイズ・保存先を制限している — 任意コード実行を防ぐため
- [ ] 外部コマンド実行にユーザー入力を渡していない — OSコマンドインジェクションを防ぐため
- [ ] リダイレクト先URLを検証している — オープンリダイレクトを防ぐため

## データ保護

- [ ] 個人情報は必要最小限しか収集・保存していない — 漏洩時の被害を最小化するため
- [ ] 秘密情報(APIキー等)は環境変数やシークレット管理で扱い、コードに書いていない — リポジトリ経由の流出を防ぐため
- [ ] 通信はすべてHTTPS — 盗聴・改ざんを防ぐため
- [ ] バックアップを定期的に取り、復元手順を確認済み — ランサムウェアや事故からの復旧のため

## 運用・監視

- [ ] エラー画面にスタックトレースや内部情報を出していない — 攻撃者への情報提供を防ぐため
- [ ] 重要操作(ログイン、権限変更、削除)のログを残している — インシデント調査のため
- [ ] ログにパスワードや個人情報を書き込んでいない — ログ経由の漏洩を防ぐため
- [ ] 依存パッケージの更新を定期的に確認する運用がある — 既知脆弱性の放置を防ぐため
- [ ] 使っていない機能・エンドポイント・アカウントを削除した — 攻撃面を減らすため

## インフラ

- [ ] 本番のDB・管理画面はインターネットから直接アクセスできない — 侵入経路を減らすため
- [ ] コンテナやプロセスをroot以外で動かしている — 侵害時の被害を限定するため
- [ ] デフォルトパスワードをすべて変更した — 最も簡単な侵入経路を塞ぐため
